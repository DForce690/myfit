# MyFit (마이핏)

> 체형 맞춤 스타일링 코치 — 내 몸에 맞는 실루엣을 설계하다

신체 치수를 입력하면 **체형을 진단**하고, 그 체형에서 가장 좋아 보이는 **코디를 근거와 함께 추천**하는 규칙 기반 스타일링 도구입니다.

**🔗 데모:** `https://github.com/DForce690/myfit/tree/main`

---

## 왜 만들었나

옷을 잘 입는다는 건 비싼 옷·예쁜 옷의 문제가 아니라, **입는 사람의 골격·비율과 옷의 실루엣이 맞느냐**의 문제입니다. 그런데 대부분의 코디 추천은 체형을 무시하거나("오버핏이 유행"), 어깨가 넓은 사람에겐 오히려 독이 되는 일반론을 줍니다. 특히 상·하의를 둘 다 헐렁하게 입어 부피가 두 배가 되는 **'더블 볼륨'** 같은 실수는 입어보기 전엔 알기 어렵습니다.

MyFit은 이 문제를 **신체 비율 정량화 → 체형 분류 → 체형별 실루엣 규칙 적용**의 단계적 추천 엔진으로 풉니다.

## 무엇을 하나

| 단계 | 기능 | 내용 |
|---|---|---|
| **1** | 신체 프로필 | 8개 치수 입력(cm/inch 토글, 유효성 검사) → 비율 자동 계산. 모르는 값은 키·몸무게로 추정하고 `(추정)` 표시 |
| **2** | 체형 진단 | 비율을 4개 체형(역삼각형·삼각형·직사각형·타원형)으로 분류, 경계값은 복합형 처리. 강점·보완점 + 스타일링 방향 3가지 리포트 |
| **3** | 추천 엔진 | 상황(TPO)·계절·범위를 받아 규칙에 맞는 코디 3개를 **근거 문장**과 함께 추천. 위반 조합은 경고 |

핵심 흐름: 새 사용자가 치수를 입력하면 자신의 체형 진단과 함께 "왜 이 옷을 입어야 하는지" 근거가 달린 코디 3개를 받습니다.

## 추천 엔진이 동작하는 방식

규칙은 두 종류로 나뉩니다.

- **[하드 제약] 더블 볼륨 방지** — 상의·하의 볼륨 합이 한도를 넘는 조합(오버핏 + 와이드 등)은 후보에서 **제외**. "한 번에 한 곳만 부풀린다."
- **[소프트 제약] 점수 가중치** — 체형별로 가점/감점:
  - 수직 라인 강조(오픈 카디건·세로 라인·프론트 턱) → 가점
  - 어깨 비율 보정(역삼각형이면 보트넥·각진 어깨 → 감점, 브이넥 → 가점)
  - 허리선 보정(하이웨이스트·프론트 턱 → 가점, 다리 비율이 짧을수록 가중)

후보 조합을 생성 → 하드 제약으로 필터링 → 소프트 제약 가중합으로 점수화 → 상위 3개를 근거와 함께 출력합니다. 👍/👎 피드백은 소프트 가중치를 세션 내에서 미세 조정합니다.

규칙 기반이라 추천에 **블랙박스가 없습니다.** 모든 결과에 "왜 이 조합인가"라는 근거가 따라붙습니다.

## 기술

빌드 과정 없는 **단일 `index.html`** — 추천 로직 전체가 클라이언트 사이드 바닐라 JS 규칙 엔진입니다.

- 프레임워크·서버·DB 없음. 의존성은 CDN(Pretendard, Space Mono)뿐
- 프로필·피드백 상태는 인메모리로 관리(세션 기준)
- 진단 리포트의 체형 실루엣은 입력 치수로 그리는 **파라메트릭 SVG**(설계 도면 콘셉트)
- 정적 호스팅(GitHub Pages 등)에 그대로 배포 가능

## 로드맵

- **시각화 고도화** — "추천 핏 vs 흔한 실수 핏" 나란히 비교
- **내 옷장 기반 추천** — 보유 의류를 등록해 실제 입을 수 있는 코디로 한정
- **개인화** — 누적 피드백으로 규칙 가중치 학습
- **확장** — AI 가상 피팅(내 사진에 옷 입히기), 쇼핑몰 연동

## 실행 방법

별도 설치 없이 `index.html`을 브라우저로 열면 됩니다. 또는:

```bash
python -m http.server 8000
# → http://localhost:8000
```
